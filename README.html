Description

Get_Next_Line (GNL) is a function that reads a file descriptor one line at a time. It's particularly useful for parsing files or standard input when you need to process content line by line.

The project demonstrates advanced C programming concepts including:

  • Static variables
  • File I/O operations
  • Memory management
  • Buffer management

Features

  • Reads from file descriptors (including STDIN)
  • Handles multiple file descriptors simultaneously
  • Efficient memory management
  • Configurable BUFFER_SIZE
  • Leak-free implementation

Usage

Basic function prototype:

char *get_next_line(int fd);

Example usage:

#include "get_next_line.h"

int main(void)
{
    int fd = open("file.txt", O_RDONLY);
    char *line;
    
    while ((line = get_next_line(fd)) != NULL)
    {
        printf("%s", line);
        free(line);
    }
    close(fd);
    return (0);
}

Installation

  1. Clone the repository: 
    git clone https://github.com/mouchtach/Get_Next_Line.git
  2. Compile the library: 
    cd Get_Next_Line && make

This will generate get_next_line.a static library.

Testing

To run basic tests:

make test

For comprehensive testing with 42's francinette:

fcinette --gnl

Note: The function returns NULL when there's nothing left to read or if an error occurs. Always check the return value!

Limitations

  • Designed for text files (not binary safe)
  • BUFFER_SIZE must be defined at compile time
  • Performance may degrade with extremely long lines
